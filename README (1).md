# 💸 SCAM Tip Bot

Телеграм-бот для перевода внутренних токенов **SCAM** между участниками чата.

Ответьте (reply) на сообщение пользователя и напишите `пер 0,048` — бот переведёт
ему 0,048 SCAM и пришлёт картинку:

```
        0,048
        SCAM
   ─────────────────
#SCAM отправил(а) 0,048 SCAM для @user
```

> ⚠️ **Важно:** SCAM здесь — это **внутренние очки**, которые бот хранит в своей
> базе данных. Это **не криптовалюта в блокчейне**, реальные монеты никуда не
> уходят и никаких кошельков/приватных ключей не требуется. Если вам нужен
> перевод **реального** on-chain токена — это другая, гораздо более сложная и
> чувствительная задача (кошельки, приватные ключи, комиссии сети). Скажите —
> помогу отдельно.

---

## Возможности

- `пер 0,048` в ответ на сообщение → перевод SCAM + картинка
- Поддержка запятой и точки: `пер 0,048` и `пер 0.048`
- `/start` — приветствие и справка
- `/balance` — ваш баланс SCAM
- `/top` — топ держателей
- `/give 100` (в ответ на сообщение, только для админов) — начислить SCAM
- База: PostgreSQL на Railway, SQLite — локально

## 🚀 Быстрый старт: GitHub + Railway

### Шаг 1. Получите токен бота

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram.
2. `/newbot` → задайте имя и username.
3. Скопируйте **токен** вида `123456789:AAE...`.

### Шаг 2. Залейте код на GitHub

```bash
cd scam-tip-bot
git init
git add .
git commit -m "SCAM tip bot"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/scam-tip-bot.git
git push -u origin main
```

### Шаг 3. Разверните на Railway

1. Зайдите на [railway.app](https://railway.app) → **New Project**.
2. **Deploy from GitHub repo** → выберите ваш репозиторий `scam-tip-bot`.
3. Railway сам увидит Python и начнёт сборку (через Nixpacks).

### Шаг 4. Добавьте базу PostgreSQL

1. В проекте нажмите **+ New** → **Database** → **Add PostgreSQL**.
2. Railway автоматически создаст переменную `DATABASE_URL` и передаст её боту.
   (Ничего вручную вписывать не нужно — код сам её подхватит.)

### Шаг 5. Задайте переменные окружения

Откройте сервис бота → вкладка **Variables** → добавьте:

| Переменная         | Значение                          | Обязательно |
|--------------------|-----------------------------------|:-----------:|
| `BOT_TOKEN`        | токен от BotFather                | ✅ да       |
| `ADMIN_IDS`        | ваш Telegram id (для `/give`)     | ⬜ нет      |
| `STARTING_BALANCE` | стартовый баланс, напр. `1000`    | ⬜ нет      |

> Свой Telegram id можно узнать у [@userinfobot](https://t.me/userinfobot).

### Шаг 6. Запуск

Railway пересоберёт и запустит сервис. В логах (**Deployments → View Logs**)
должно появиться `Запускаю polling…`. Всё — напишите боту `/start`.

## 🖥️ Локальный запуск (для проверки)

```bash
git clone https://github.com/ВАШ_ЛОГИН/scam-tip-bot.git
cd scam-tip-bot
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env        # впишите свой BOT_TOKEN
export $(cat .env | xargs)  # Windows: задайте переменные вручную
python bot.py
```

Без `DATABASE_URL` бот использует локальный файл `scam.db` (SQLite).

## ⚙️ Как это работает

| Файл           | Назначение                                              |
|----------------|---------------------------------------------------------|
| `bot.py`       | логика бота, команды и обработчик `пер …`               |
| `db.py`        | база данных (SQLAlchemy async), балансы и переводы       |
| `image_gen.py` | генерация PNG-картинки перевода (Pillow)                 |
| `assets/`      | шрифты DejaVu (с поддержкой кириллицы)                   |
| `requirements.txt` / `Procfile` / `railway.json` / `.python-version` | сборка и деплой |

Триггер перевода — регулярное выражение `^\s*пер\s+(\d+(?:[.,]\d+)?)\s*$`.
Сообщение обязательно должно быть **ответом** на чьё-то сообщение — его автор и
становится получателем.

## 🎨 Настройка

- **Текст на картинке** и цвета — в `image_gen.py` (переменные `ACCENT`, `BG_*`,
  функция `render_transfer_image`).
- **Триггер** `пер` — в `bot.py`, переменная `TRANSFER_RE`.
- **Стартовый баланс** — переменная окружения `STARTING_BALANCE`.

## ❓ Частые вопросы

**Бот не отвечает.** Проверьте `BOT_TOKEN` в Variables и логи Railway.

**«Недостаточно средств».** У отправителя закончились SCAM — начислите через
`/give` (от админа) или увеличьте `STARTING_BALANCE`.

**Данные пропали после передеплоя.** Значит используется SQLite без тома —
подключите PostgreSQL (Шаг 4): файловая система Railway эфемерная.
